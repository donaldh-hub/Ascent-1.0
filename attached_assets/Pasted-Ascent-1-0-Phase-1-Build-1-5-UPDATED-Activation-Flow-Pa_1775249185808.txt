PATCH REQUIREMENTS

==================================================

1. FIRST-RUN SETUP GATING

On authenticated login, check setup completion:

Setup is COMPLETE only if:

* organization exists
* at least one property exists
* at least one unit exists

If NOT complete:

* force redirect to /setup
* block access to dashboard, analytics, workflows, etc.

If complete:

* allow normal app routing

This must apply on:

* login
* page refresh
* direct URL navigation

==================================================

2. SETUP COMPLETION STATE

Implement reliable completion detection.

Preferred:

* derive from real data (properties + units)

Optional (if needed):

* add organization field: is_setup_complete (boolean)
* update to true ONLY after successful setup completion

Do NOT allow manual toggling without real data.

==================================================

3. SETUP ROUTE + FLOW

Create route:

* /setup

This route must be a guided, step-by-step flow.

Steps:

Step 1 — Welcome

* “Let’s set up your system”
* Button: Start Setup

Step 2 — Property Creation

* Create first property
* Fields:

  * property name (required)
  * address (optional)

Step 3 — Unit Intake Choice
User must choose:

* Upload CSV
* Manual Entry

Step 4 — Unit Creation

* CSV path:

  * upload file
  * map fields
  * preview
  * confirm import
* Manual path:

  * add units quickly
  * minimal friction input

Step 5 — Completion

* show summary:

  * properties created
  * units created
* message: “Your system is now active”
* CTA: Go to Units

==================================================

4. REDIRECTION AFTER SETUP

After completion:

* redirect to /units
* mark setup as complete (derived or stored)

User must NOT be redirected back to /setup after completion.

==================================================

5. PROTECTED ROUTES UPDATE

Update route guard logic:

If setup incomplete:

* redirect ALL protected routes → /setup

Exceptions:

* /setup
* logout/auth routes

==================================================

6. UX REQUIREMENTS

Setup must feel:

* fast
* simple
* guided
* confidence-building

User must experience:

* immediate progress
* clear next step
* no confusion about where to start

==================================================

7. EMPTY STATE HANDLING

If user has:

* no properties
* no units

Do NOT show dashboard placeholders.
Redirect to setup immediately.

==================================================

8. AUDIT + TRACKING

Track:

* setup started
* property created
* units created
* setup completed

==================================================

9. SUCCESS STATE

This patch is complete when:

1. New user logs in
2. User is forced into /setup
3. User creates property
4. User creates or uploads units
5. User completes setup
6. User is redirected to /units
7. User can now access full app normally
8. Refresh does NOT send user back to setup

==================================================

This is ACTIVATION ONLY.

==================================================
CORE CALCULATIONS
=================

1. TURN COMPLETION %

Calculate completion using weighted stage progress.

Example structure:

* Each stage contributes proportionally
* Completed stages = full weight
* Current stage = partial weight
* Future stages = 0

Output:

* completion_percentage (0–100)

---

2. STAGE AGING

Calculate:

* days_in_stage
* total_days_open

Outputs:

* aging severity
* longest-aging stage
* aging rank per turn

---

3. BLOCKED TURN DETECTION

A turn is BLOCKED if:

* days_in_stage exceeds defined threshold
* OR no movement detected within threshold window

Output fields:

* is_blocked (true/false)
* blocked_stage
* blocked_duration

---

4. REWORK LOGIC

If:

* inspection fails OR rework_required = true

Then:

* activate Rework stage
* reduce effective completion % confidence
* flag as execution risk

Outputs:

* rework_flag
* rework_count (if repeat)

---

5. RENT-READY STATUS

A turn is RENT READY only if:

* completion = 100%
* inspection_passed = true
* no active rework

Output:

* rent_ready (true/false)

==================================================
BOTTLENECK DETECTION (CRITICAL)
===============================

Aggregate across all turns:

Identify:

* stage with highest volume
* stage with highest average aging
* stage with most blocked turns

Outputs:

* primary_bottleneck_stage
* bottleneck_severity_score
* bottleneck_explanation

Example:

“Flooring is the primary bottleneck due to high aging and blocked volume.”

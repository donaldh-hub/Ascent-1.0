BUILD PURPOSE

Build 7.2 adds the analysis layer that sits after Build 7.1 reporting ingestion.

Build 7 created the reporting and analytics backbone.

Build 7.1 created the trusted reporting intake gate and classified normalized reporting records as:

1. Fully Reportable
2. Partially Reportable
3. Not Reportable Yet

Build 7.2 must now analyze only eligible reporting records and produce operational intelligence around:

1. Where time is going
2. Where labor capacity is being consumed
3. Where delays are forming
4. Which categories are creating operational drag
5. Which properties, units, workflows, turns, work orders, PM items, assets, or evidence gaps are contributing to that drag

7. Which Control Tower, Reports, and future Snapshot surfaces should be able to consume this analysis

REQUIRED DATA HANDLING RULES

1. Fully Reportable records may be used in confirmed analysis.

2. Partially Reportable records may be used only in qualified analysis.

3. Not Reportable Yet records must not be used in final metrics, trends, bottleneck conclusions, or Control Tower scores.

4. Every analysis result must carry a confidence/truth classification.

Use these analysis confidence states:

- Confirmed Analysis
- Qualified Analysis
- Insufficient Data

5. Confirmed Analysis may only be generated from Fully Reportable records.

6. Qualified Analysis may include Partially Reportable records but must clearly label what is missing.

7. Insufficient Data must be used when the system does not have enough reportable information to support a conclusion.

8. Do not silently mix Fully Reportable and Partially Reportable records.

9. Do not create averages, percentages, or time allocation claims if the underlying fields are missing.

10. If date fields are missing, do not calculate cycle time.

11. If category fields are missing, do not calculate category share.

12. If property/unit assignment is missing, do not attribute the record to a property/unit as fact.

13. If responsible party data is missing, do not calculate staff/contractor split as fact.

14. If supporting records are unavailable, do not surface the analysis as clickable proof.

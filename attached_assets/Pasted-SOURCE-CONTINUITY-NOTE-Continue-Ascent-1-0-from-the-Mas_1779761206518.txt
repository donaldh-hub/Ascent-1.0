MANDATORY ACTIVE REPORTING MODE REQUIREMENT

Build 7.4 must preserve and visually respect the active reporting mode logic created in Build 7.2.1 and clarified in Build 7.3.1.

Before reporting turn performance, turn visuals, turn narrative insights, or turn-related dashboard signals, the system must recognize the organization’s selected reporting mode:

1. Separate Turn Mode

Used when the organization tracks turns/make-ready work separately from work orders.

In this mode:

- Turn reports must use turn records.
- Turn visuals must use turn stages.
- Turn bottlenecks must reference make-ready/turn stages.
- Work orders may be shown as related supporting context only when linked.
- Work orders must not be treated as the primary measurement of turn progress.

2. Work-Order-Measured Turn Mode

Used when the organization uses work orders as the practical measurement of turn progress.

In this mode:

- Turn performance may be derived from linked work orders.
- Turn progress visuals must clearly explain that work orders are being used as the progress signal.
- The system must distinguish between general work orders and work orders connected to a turn/make-ready process.
- Turn reports must not silently mix all work orders into turn reporting.
- Supporting records must show why each work order was included in the turn report.

3. Unknown / Not Configured Mode

Used when the organization has not selected or provided enough information to determine how turns are tracked.

In this mode:

- Do not show confident turn performance conclusions.
- Show an honest configuration-needed state.
- Explain that turn reporting depends on whether the organization separates turn records or uses work orders to measure turn progress.
- Prompt the user to confirm their turn reporting mode before relying on turn metrics.


TURN VISUAL REQUIREMENTS

Every turn-related visual must display or respect the active reporting mode.

Required user-facing clarity:

If Separate Turn Mode is active, show language such as:

“Turn performance is based on dedicated turn/make-ready records.”

If Work-Order-Measured Turn Mode is active, show language such as:

“Turn performance is being measured through work orders linked to the turn process.”

If Unknown / Not Configured Mode is active, show language such as:

“Turn reporting mode has not been confirmed yet. Confirm whether your organization tracks turns separately or uses work orders to measure turn progress.”

No turn visual may hide the active mode.

No turn visual may silently blend general work orders into turn performance.


TURN SUPPORTING RECORD REQUIREMENTS

Every turn signal must open supporting records that match the active mode.

If Separate Turn Mode is active:

- Supporting records must be turn records or turn-stage records.
- Linked work orders may appear only as related context, not as the primary turn measurement.

If Work-Order-Measured Turn Mode is active:

- Supporting records must be only the work orders linked to turn progress.
- Each record must show the reason it was included.
- General unrelated work orders must not appear in turn supporting records.

If Unknown / Not Configured Mode is active:

- Do not create clickable turn certainty signals.
- Show configuration guidance instead.

DATA CONFIDENCE REQUIREMENTS

Every visual block must carry one of these data confidence states:

1. Confirmed
2. Partial
3. Limited
4. Not available

Use these rules:

Confirmed:

Only fully reportable / fully resolved records are used.

Partial:

Some records are reportable, but partial records are included with clear labeling.

Limited:

The system has too little complete data to make a strong conclusion.

Not available:

The required source records do not exist or are not resolved enough to report.

Do not show strong conclusions for limited data.

Do not display partial metrics as executive certainty.


EMPTY / LOW-DATA STATE REQUIREMENTS

Every report and visual block must have honest empty states.

Examples:

1. “No fully reportable work order records are available for this date range.”
2. “Some records were uploaded, but they are not reportable yet because property or unit assignment is missing.”
3. “Trend analysis requires dated records across more than one reporting period.”
4. “Time allocation cannot be calculated because duration, start date, completion date, or labor fields are missing.”
5. “This report is limited because unresolved records were excluded from confirmed metrics.”
6. “Turn reporting mode has not been confirmed yet. Confirm whether your organization tracks turns separately or uses work orders to measure turn progress.”

Do not show blank panels.

Do not show fake charts.

Do not show placeholder numbers.

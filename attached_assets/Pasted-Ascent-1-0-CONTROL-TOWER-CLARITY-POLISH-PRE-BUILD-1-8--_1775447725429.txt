CORE GOAL
=========

Make the system explain itself instantly.

* WHY it is happening

==================================================
UPDATE 1 — PRIMARY CAUSE LINE
=============================

For every diagnostic section (Flow, Risk, Execution, Improvement):

Add a top line labeled:

"PRIMARY CAUSE"

This must:

* summarize the biggest driver
* be one clear sentence
* use real data

Example:
"Approval bottleneck with 84 items aging >14 days"

Place this ABOVE all other diagnostic content.

==================================================
UPDATE 2 — NUMBER CONTEXT LABELS
================================

All major numbers must include context labels.

Examples:

* "647 critical items (HIGH — requires immediate attention)"
* "206 overdue (SEVERE backlog)"
* "0% completion (CRITICAL — no progress)"

Rules:

* use HIGH / MODERATE / LOW / SEVERE / CRITICAL
* derive label from existing scoring thresholds
* DO NOT invent new scoring logic

==================================================
UPDATE 3 — RECOMMENDED ACTION BLOCK
===================================

At the bottom of every diagnostic section add:

"RECOMMENDED ACTION"

This must:

* be 1–2 lines
* based on actual system signals
* point to immediate next step

Example:
"Review approval delays and resolve top 3 aging items first"

==================================================
UPDATE 4 — DIAGNOSTIC VISUAL EMPHASIS
=====================================

Make diagnostic sections visually distinct from standard cards.

Apply ONE OR MORE:

UPDATE 5 — TOP-LEVEL WHY (CRITICAL)
===================================

"DRIVEN BY"

This must always reflect REAL data.

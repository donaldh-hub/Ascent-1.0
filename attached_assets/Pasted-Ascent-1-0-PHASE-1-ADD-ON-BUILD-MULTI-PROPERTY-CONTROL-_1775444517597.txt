NEW SERVICE LAYER
=================

Create a new service:

portfolio_control_tower_service

Responsibilities:

* aggregate property-level data
* reuse scoring engine outputs
* reuse alert signals
* reuse evidence signals
* generate ranked property summaries
* generate communication-ready summaries (email content)

Keep logic OUT of UI components.

==================================================
PROPERTY-LEVEL AGGREGATION
==========================

For each property compute:

* health_score (0–100)

* stoplight (green/yellow/red)

* flow_score

* risk_score

* improvement_score

* execution_score

* critical_items_count

* top_bottleneck (stage/workflow)

* bottleneck_aging

* missing_docs_count (CRITICAL only)

* document_count (optional)

* trend_direction (if available)

* supervisor_name

* supervisor_email

Return shape:

{
property_id,
property_name,
health_score,
stoplight,
flow_score,
risk_score,
improvement_score,
execution_score,
critical_items_count,
top_bottleneck,
bottleneck_aging,
missing_docs_count,
document_count,
trend_direction,
supervisor_name,
supervisor_email,
insight_summary,
communication_summary
}

==================================================
PRIORITIZATION ENGINE
=====================

Sort properties by urgency:

1. RED first
2. then YELLOW
3. then GREEN

Within group sort by:

* highest risk
* highest aging
* highest critical_items_count

DO NOT sort alphabetically.
